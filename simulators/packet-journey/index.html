What the terms mean

  • Hop: one link between two nodes (client → router → server).
  • Latency: the base delay each hop adds to a packet.
  • Jitter: randomness on top of latency; causes spread in arrival times.
  • Loss per hop: chance a packet disappears on that hop.
  • Delivery rate: delivered ÷ created. Loss compounds across hops.
  • Latency histogram: shows how spread out arrival times are.
  • Per-hop loss chips: where drops happened most often.

How changes affect the sim

  • Higher latency → all packets arrive later; histogram shifts right.
  • Higher jitter → histogram widens; p95 pulls away from p50.
  • Higher loss → more red dots; per-hop chips count where drops occur.
  • More packets → smoother stats and clearer patterns.
  • Replay a run to see identical timing versus a new run's randomness.

Try these

  • Set loss to 0%, jitter to 0 → a tight, green delivery line.
  • Jitter 300ms, latency 80ms → wide histogram, p95 much higher than p50.
  • Loss 20%, 40 packets → watch per-hop loss chips spike and delivery rate drop.
  • Replay the same run to see deterministic positions vs a fresh run's randomness.
